# 별내 맛집 가이드 v2 — 갈매 확장 + 지도/오늘의픽 설계

작성일: 2026-07-18
대상 파일: `index.html` (= `korea-bar-guide.html` 사본, 단일 HTML, dep 0)

## 목표

기존 "별내 근처 맛집 가이드"(별내 100 + 회기 21 + 석계 20 = 141곳)를
**별내 + 갈매 2지역 총 150곳** 구성으로 재편하고, 지역 네비를 경량 SVG
인터랙티브 지도로 바꾸고, "오늘의 픽" 랜덤 버튼을 추가한다.

## 확정 결정

- **회기·석계 완전 삭제** (총 41곳 DATA 제거)
- **별내 100 유지 + 갈매 50 신규 = 150곳**
- 갈매 50곳은 **네이버 플레이스 실데이터 리서치**로 채운다 (별내와 동일 수준)
- 지도: 텍스트 city-row → **경량 인라인 SVG 2구역맵**(별내·갈매), API키·좌표 불필요
- 오늘의 픽 버튼 위치: **지도밴드(AREA NAVI) 안**
- 오늘의 픽 스코프: **전체 150곳에서 3곳 랜덤**
- 타이틀/히어로: **"위대한 AI님의 별내 근처 맛집 가이드" 유지** ('근처' 유지)

## 1. 데이터 (최대 작업)

### 삭제
- `DATA`에서 `city:'회기'`(21) + `city:'석계'`(20) 항목 전부 제거.

### 갈매 50곳 신규
- 갈매동(구리시 갈매지구) 신도시 상권. 밥집+술집 혼합, 별내 배분 감각 따름.
- 항목 스키마(별내와 동일):
  `{city:'갈매', name, bucket, genre, address, access, budget, closed?,
    score?, visitorReviews, blogReviews, naverUrl, menu:[문자열3],
    photo, extraPhotos:[3], card?, reserve?}`
- 각 항목 실측: 네이버 플레이스에서 방문자리뷰수·블로그리뷰수·평점(score)·
  주소·메뉴3·대표사진+리뷰사진3·플레이스 URL. `insane-search`로 네이버 접근.
- bucket 값은 기존 카테고리 재사용(고기구이/면·국수/이자카야/파스타·양식 등).
  새 bucket 필요시 추가 — 업종칩은 DATA에서 동적 생성이라 자동 반영.
- 선별 기준: 네이버 방문자리뷰 상위 + 블로그 교차검증(별내와 동일).

### 리서치 리스크
- 50곳 실데이터 = 다수 네이버 조회. 사진 URL은 `pstatic.net` 직링크로
  실제 로드되는 것만 사용(깨진 URL 금지). 리뷰수는 스냅샷—리서치 기준일 명기.

## 2. 지역 로직

- `const CITY_ORDER = ['별내','갈매'];` (기존 `['별내','회기','석계']` 교체)
- 도시탭·업종칩·리뷰수필터·정렬·카운트는 CITY_ORDER/DATA 기반 동적 → 자동 반영.
  별도 수정 불필요(확인만).
- 별점: `item.score` 있으면 표시(도시 무관). 갈매 항목에 score 포함.

## 3. 지도 (경량 SVG 2구역맵)

- 위치: 기존 `<section class="map-band">` 내부.
- `map-city-row` 3개 텍스트 행 제거 → 인라인 `<svg>` 2구역:
  - 별내 존(좌/상), 갈매 존(우/하) — 스키매틱 배치(지리 정밀도 아님).
  - 각 존: `<path>`/`<rect>` + 라벨(한글+로마자+곳수).
  - 존 클릭 → 기존 `pickCity(city)` 호출(도시탭 click → 필터 → 스크롤).
    `pickCity`는 그대로 재사용.
- `initMapNav()` 수정: `.map-city-row` 대신 SVG 존(`[data-city]`)에 리스너.
  곳수 동적 반영 로직(`counts`)은 SVG 라벨 대상으로 유지.
- 반응형: `viewBox` + `max-width:100%`. 라이트/다크 모두 대비 확보.

## 4. 오늘의 픽 (신규)

- 지도밴드 안 버튼: `🎲 오늘의 픽`.
- 클릭 → 전체 `DATA`(150)에서 3곳 랜덤 추출(중복 없이) → 미니 패널에 카드 3장.
  - 카드: 이름(네이버링크)·지역·업종·대표사진·평점. 기존 카드 스타일 재사용/축약.
  - "다시 뽑기" 버튼으로 재추출.
  - 카드 클릭 시 해당 매장으로 스크롤 or 네이버 링크(간단히 네이버 링크).
- 순수 JS, 외부 의존 없음.

## 5. 텍스트 정리

- 스키마 주석: `city:'회기'|'석계'` → `city:'별내'|'갈매'`.
- 범례(legend): 회기·석계 문구 삭제 → "별내·갈매" 기준 설명으로.
- 푸터: 회기·석계 선별기준 문단 제거/재작성, 리서치 기준일에 갈매(2026-07-18) 추가.
- 히어로 `<h2>`/`heroCount`: "2개 지역 총 150곳"으로 동적 표기(코드가 자동).
